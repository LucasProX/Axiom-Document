# 2.2 新建密钥

> 目标：生成一把用于客户端鉴权的 API Key，并妥善保存。

## 一、点击「创建密钥」

在密钥列表页，点击顶部的「创建密钥 / 新建 Key」按钮。

## 二、填写密钥信息

| 字段 | 必填 | 说明 |
| --- | :---: | --- |
| **名称** | 是 | 给这把 Key 起个一眼能认出来的名字，如 `mac-codex`、`ci-pipeline`，方便日后管理 |
| **额度上限** | 否 | 为单把 Key 设置消费上限；超出后该 Key 自动停用，不影响主账户余额 |
| **可用模型** | 否 | 限制此 Key 能访问的模型（如只允许 Claude Sonnet / Opus） |
| **过期时间** | 否 | 设置过期时间，适合临时分享的场景 |

![填写密钥信息](/images/apikey/05-fill-key-form.png)

## 三、提交并复制 Key

点击「确认创建」，立即把显示的密钥（格式 `sk-XXXX...`）复制到密码管理器中。

::: danger 只显示一次
完整字符串关闭后**无法再次查看**。如果忘了复制，只能删除重建。
:::

![复制密钥](/images/apikey/06-copy-key.png)

## 四、回到列表页确认

关闭弹窗，新建的 Key 会出现在列表中。

![列表中的新密钥](/images/apikey/07-key-list-after.png)

::: tip 安全建议
按设备 / 项目 / 成员分别创建不同的 Key，并各自设置「额度上限」。这样某把 Key 泄露时，可单独停用且损失可控。
:::

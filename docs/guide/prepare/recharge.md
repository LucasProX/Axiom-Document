# 1.2 充值与订阅

> AxiomCode 采用「预付额度、按量计费」：账户里有多少余额，就能用多少 token。**没有自动订阅、不强制绑卡。**

## 什么时候充值

按使用强度，给个大致参考：

| 强度 | 金额 | 典型场景 |
| --- | --- | --- |
| 轻度试用 | ¥5 – ¥10 | 每天写写代码片段、跑跑 demo |
| 个人主力 | ¥30 – ¥80 / 月 | 日常用 Codex / Claude Code 做真实项目 |
| 团队 / 重度 | ¥200+ | 多台机器、多客户端、CI 自动化、长上下文 |

## 进入充值页

登录后点击钱包图标，或直接访问 **[https://axiomcode.top/billing](https://axiomcode.top/billing)**。页面会显示当前余额、历史消费与充值入口。

![充值入口](/images/prepare/06-billing-entry.png)

## 两种充值方式

### 方式 A：兑换码（推荐）

1. 访问小店 [https://pay.ldxp.cn/shop/axiom](https://pay.ldxp.cn/shop/axiom)。
2. 选择面额、付款，立即获得兑换码。
3. 回到充值页粘贴兑换码，余额即时到账。

详细步骤见 [1.3 使用兑换码](/guide/prepare/redeem)。

### 方式 B：在线支付

在充值页直接通过微信 / 支付宝充值（是否可用取决于账户所在地区）。

## 注意事项

![充值记录](/images/prepare/07-recharge-history.png)

::: tip 到账确认
充值成功后，新的明细会出现在「充值记录」中。若几分钟后余额仍未更新，刷新页面，或携带订单 / 兑换码截图联系客服。
:::

::: warning 余额不足会被拒
当余额低于 ¥1 时，所有 API 调用都会被拒绝并返回「额度不足」错误。建议提前充值，或开启低余额提醒（如有）。
:::
